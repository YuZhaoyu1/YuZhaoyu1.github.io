453. 最小操作次数使数组元素相等

题目要求

给你一个长度为 n 的整数数组，每次操作将会使 n - 1 个元素增加 1 。返回让数组所有元素相等的最小操作次数。

示例 1：

输入：nums = [1,2,3]
输出：3
解释：
只需要3次操作（注意每次操作会增加两个元素的值）：
[1,2,3]  =>  [2,3,3]  =>  [3,4,3]  =>  [4,4,4]

示例 2：

输入：nums = [1,1,1]
输出：0

提示：

  • n == nums.length
  • 1 <= nums.length <= 105
  • -109 <= nums[i] <= 109
  • 答案保证符合 32-bit 整数

个人解答

  1. 使 n - 1 个元素增加 1可以等价于使一个元素减少1，这样只需要将每个元素减少到数组中的最小的元素的值即可
  2. 将数组中所有元素都减少到最小元素所花费的步数加起来就是最少的步数
class Solution {
    public int minMoves(int[] nums) {
        // 步骤1: 找到数组中的最小值
        int min = Integer.MAX_VALUE;
        for (int num : nums) {
            if (num < min) {
                min = num;
            }
        }

        // 步骤2: 计算差值并累加
        int operations = 0;
        for (int num : nums) {
            operations += (num - min);
        }

        return operations;
    }
}

665. 非递减数列

题目要求：

给你一个长度为 n 的整数数组 nums ，请你判断在 最多 改变 1 个元素的情况下，该数组能否变成一个非递减数列。

我们是这样定义一个非递减数列的： 对于数组中任意的 i (0 <= i <= n-2)，总满足 nums[i] <= nums[i + 1]。

示例 1:

输入: nums = [4,2,3]
输出: true
解释: 你可以通过把第一个 4 变成 1 来使得它成为一个非递减数列。

示例 2:

输入: nums = [4,2,1]
输出: false
解释: 你不能在只改变一个元素的情况下将其变为非递减数列。

提示：

  • n == nums.length
  • 1 <= n <= 104
  • -105 <= nums[i] <= 105

个人解答

class Solution {
    public boolean checkPossibility(int[] nums) {
        int a = 0;
        for(int i=0;i<nums.length-1;i++){
            if(nums[i] > nums[i+1]){
                a++;
                if(i == 0){
                    nums[i] = Integer.MIN_VALUE;
                }
                else{
                    if(nums[i-1]>nums[i+1]){
                        nums[i+1] = nums[i];
                    }
                    else{
                        nums[i] = nums[i-1];
                    }
                }
                if(a>1){
                    return false;
                }
            }
        }
        return true;
    }
}
  1. 遍历数组，判断是否非递减，用a来记录操作次数
  2. 如果nums[i] > nums[i+1]，则说明要进行操作
  3. 如果这时是数组的第一个元素，则操作是将其变得极小或者让nums[1]=nums[0]
  4. 如果不是第一个元素，对当前元素的操作只能是将当前元素变成前一个元素或者将下一个元素变成当前元素，比较i-1个元素和i+1个元素的大小，如果i-1个元素比i+1个元素大的话，只能将下一个元素变成当前元素；反之就只能将当前元素变成前一个元素

283. 移动零

题目要求

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

请注意 ，必须在不复制数组的情况下原地对数组进行操作。

示例 1:

输入: nums = [0,1,0,3,12]
输出: [1,3,12,0,0]

示例 2:

输入: nums = [0]
输出: [0]

提示:

  • 1 <= nums.length <= 104
  • -231 <= nums[i] <= 231 - 1

个人解答

class Solution {
    public void moveZeroes(int[] nums) {
        int count = 0;
        for(int i=0;i<nums.length;i++){
            if (nums[i] == 0){
                count++;
            }
            else{
                nums[i-count] = nums[i];
            }
        }
        for(int i=0;i<count;i++){
            nums[nums.length-count+i] = 0;
        }
    }
}

189. 轮转数组

题目要求

给定一个整数数组 nums，将数组中的元素向右轮转 k 个位置，其中 k 是非负数。

示例 1:

输入: nums = [1,2,3,4,5,6,7], k = 3
输出: [5,6,7,1,2,3,4]
解释:
向右轮转 1 步: [7,1,2,3,4,5,6]
向右轮转 2 步: [6,7,1,2,3,4,5]
向右轮转 3 步: [5,6,7,1,2,3,4]

示例 2:

输入：nums = [-1,-100,3,99], k = 2
输出：[3,99,-1,-100]
解释: 
向右轮转 1 步: [99,-1,-100,3]
向右轮转 2 步: [3,99,-1,-100]

提示：

  • 1 <= nums.length <= 105
  • -231 <= nums[i] <= 231 - 1
  • 0 <= k <= 105

个人解答

class Solution {
    public void rotate(int[] nums, int k) {
        int[] a = new int[nums.length];
        k = k%nums.length;
        for(int i=0;i<nums.length;i++){
            if(i<k){
                a[i] = nums[nums.length-k+i];
            }
            else{
                a[i] = nums[i-k];
            }
        }
        for(int i=0;i<nums.length;i++){
            nums[i] = a[i];
        }
    }
}

396. 旋转函数

题目要求

给定一个长度为 n 的整数数组 nums 。

假设 arrk 是数组 nums 顺时针旋转 k 个位置后的数组，我们定义 nums 的 旋转函数 F 为：

  • F(k) = 0 * arrk[0] + 1 * arrk[1] + ... + (n - 1) * arrk[n - 1]

返回 F(0), F(1), ..., F(n-1)中的最大值 。

生成的测试用例让答案符合 32 位 整数。

示例 1:

输入: nums = [4,3,2,6]
输出: 26
解释:
F(0) = (0 * 4) + (1 * 3) + (2 * 2) + (3 * 6) = 0 + 3 + 4 + 18 = 25
F(1) = (0 * 6) + (1 * 4) + (2 * 3) + (3 * 2) = 0 + 4 + 6 + 6 = 16
F(2) = (0 * 2) + (1 * 6) + (2 * 4) + (3 * 3) = 0 + 6 + 8 + 9 = 23
F(3) = (0 * 3) + (1 * 2) + (2 * 6) + (3 * 4) = 0 + 2 + 12 + 12 = 26
所以 F(0), F(1), F(2), F(3) 中的最大值是 F(3) = 26 。

示例 2:

输入: nums = [100]
输出: 0

提示:

  • n == nums.length
  • 1 <= n <= 105
  • -100 <= nums[i] <= 100

个人解答

  1. 设数组nums的元素之和是count，长度为n

  2. F(0)=0×nums[0]+1×nums[1]+…+(n−1)×nums[n−1]

    F(1)=1×nums[0]+2×nums[1]+…+(n−1)×nums[n−2]+0×nums[n−1]

    F(2)=2×nums[0]+3×nums[1]+…+(n−1)×nums[n−3]+0×nums[n−2]+1×nums[n−1]

    ……….

  3. 据此可以推出公式：F(k)=F(k−1)+count−n×nums[n−k]，1≤k<n

class Solution {
    public int maxRotateFunction(int[] nums) {
        int count = 0;
        int sum = 0;
        for(int i=0;i<nums.length;i++){
            sum += i*nums[i];
            count += nums[i];
        }
        int max = sum;
        for(int i=nums.length-1;i>0;i--){
            sum += count-nums.length*nums[i];
            if(sum>max){
                max = sum;
            }
        }
        return max;
    }
}

645. 错误的集合

题目要求

集合 s 包含从 1 到 n 的整数。不幸的是，因为数据错误，导致集合里面某一个数字复制了成了集合里面的另外一个数字的值，导致集合 丢失了一个数字 并且 有一个数字重复 。

给定一个数组 nums 代表了集合 S 发生错误后的结果。

请你找出重复出现的整数，再找到丢失的整数，将它们以数组的形式返回。

示例 1：

输入：nums = [1,2,2,4]
输出：[2,3]

示例 2：

输入：nums = [1,1]
输出：[1,2]

提示：

  • 2 <= nums.length <= 104
  • 1 <= nums[i] <= 104

个人解答

class Solution {
    public int[] findErrorNums(int[] nums) {
        int[] result = new int[2];
        int len = nums.length;
        int[] a = new int[len+1];
        for(int i=0;i<len;i++){
            a[nums[i]]++;
        }
        for(int i=1;i<=len;i++){
            if(a[i] == 0){
                result[1] = i;
            }
            if(a[i] == 2){
                result[0] = i;
            }
        }
        return result;
    }
}

1991. 找到数组的中间位置

题目要求

给你一个下标从 0 开始的整数数组 nums ，请你找到 最左边 的中间位置 middleIndex （也就是所有可能中间位置下标最小的一个）。

中间位置 middleIndex 是满足 nums[0] + nums[1] + ... + nums[middleIndex-1] == nums[middleIndex+1] + nums[middleIndex+2] + ... + nums[nums.length-1] 的数组下标。

如果 middleIndex == 0 ，左边部分的和定义为 0 。类似的，如果 middleIndex == nums.length - 1 ，右边部分的和定义为 0 。

请你返回满足上述条件 最左边 的 middleIndex ，如果不存在这样的中间位置，请你返回 -1 。

示例 1：

输入：nums = [2,3,-1,8,4]
输出：3
解释：
下标 3 之前的数字和为：2 + 3 + -1 = 4
下标 3 之后的数字和为：4 = 4

示例 2：

输入：nums = [1,-1,4]
输出：2
解释：
下标 2 之前的数字和为：1 + -1 = 0
下标 2 之后的数字和为：0

示例 3：

输入：nums = [2,5]
输出：-1
解释：
不存在符合要求的 middleIndex 。

示例 4：

输入：nums = [1]
输出：0
解释：
下标 0 之前的数字和为：0
下标 0 之后的数字和为：0

提示：

  • 1 <= nums.length <= 100
  • -1000 <= nums[i] <= 1000

个人解答

  1. 用num_l和num_r表示左右元素的和
  2. 先算出当下标为0时的右侧元素的和，即从num[1]到nums[n]的和，此时的左侧元素的和为0
  3. 从下标为0的元素开始遍历，判断左右元素的和是否相等，若相等则当前下标就是中间位置，若不相等则看下一个位置，同时左右元素的和也做变换
  4. 如果直到最后一个元素都不相等，就返回-1
class Solution {
    public int findMiddleIndex(int[] nums) {
        int len = nums.length;
        int sum_l = 0;
        int sum_r = 0;
        for(int i=1;i<len;i++){
            sum_r += nums[i];
        }
        for(int i=0;i<len;i++){
            if(sum_l == sum_r){
                return i;
            }
            if(i == len-1){
                return -1;
            }
            sum_l += nums[i];
            sum_r -= nums[i+1];
        }
        return 0;

    }
}
